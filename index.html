The Basics

What is Gas Town?

Gas Town is a system that lets you run many AI agents at the same time, all working on your code. Instead of one AI assistant doing one thing at a time, you have an entire team.

🎯

You give the order

Tell the Mayor what you want in plain English. "Build me a login page." "Fix these 12 bugs." "Add unit tests for the API." The Mayor is your project manager — it listens, plans, and delegates.

📋

Work gets broken down

The Mayor splits your request into individual tasks called "beads" (like tickets in Jira). Each bead is a focused, well-scoped piece of work that one agent can handle independently.

🔧

Workers build in parallel

Workers (called Polecats) each pick up a task and code it in their own isolated copy of your project. Five workers can build five features at the same time — no one steps on anyone else's toes.

⚙️

Code merges safely

When workers finish, the Refinery checks every piece of code before merging it. Conflicts are caught and resolved. Nothing broken ever lands on your main branch.

🛡️

Monitors keep everything healthy

The Witness watches every worker in each project. If one gets stuck, it flags the Mayor who reassigns the work. The Deacon runs health checks across the whole system. If an agent crashes, its work is saved and automatically resumed when it restarts.

Value Propositions

Why Gas Town?

Not just another AI coding tool. A fully managed, self-healing development team that works while you sleep.

⚡

Speed through parallelism

Instead of one agent working 2 hours, 5 agents finish in 25 minutes. Each worker codes simultaneously in their own isolated branch.

🛡️

Crash-proof persistence

If an agent crashes, its work is saved on a "hook." When it restarts, it picks up exactly where it left off. No lost work, ever.

📋

Built-in project management

Every task is tracked as a "bead" (like a Jira ticket). The Mayor creates them, assigns them, and closes them. You always know what's done and what's pending.

🔀

Automatic code merging

The Refinery checks every piece of code before it merges. Conflicts are caught and resolved. Nothing broken lands on your main branch.

💡

Self-healing workforce

The Witness monitors every worker. If one gets stuck, it flags the Mayor who reassigns the work. The Deacon runs health checks across the whole system.

📁

Works with any project

💬

Talk, don't type commands

Day to day, you just talk to the Mayor in plain English. "Build me a login page." "Fix the 12 bugs in this list." The Mayor handles the rest.

Get Running

Quickstart

From zero to a working AI workforce in five steps. Each one takes about a minute.

Install

Install Gas Town and its dependencies, then initialize your workspace.

# Install Gas Town and dependencies brew install gastown brew install tmux # Initialize your workspace gt install ~/gt --git

Start it up

Boot the entire Gas Town system with a single command.

cd ~/gt gt up

Add your project

Point Gas Town at any git repository. It becomes a managed "rig."

# Add any git repo as a "rig" gt rig add my-project ~/path/to/your/repo # Wake it up and start its agents gt rig undock my-project gt rig start my-project

Talk to the Mayor

Attach to your AI project manager and describe what you want in plain English.

# Attach to your AI project manager gt mayor attach # Now just talk in plain English: # "Build me a login page with email and password" # "Fix all the CSS spacing issues across the site" # "Add unit tests for the API endpoints"

Watch it work

Monitor your AI workforce as it builds, fixes, and ships code.

gt status # See everything running gt convoy list # Track progress on all work gt polecat list # See all active workers

In Practice

Real world examples

Three scenarios showing Gas Town in action — from a single sentence to shipped code.

Build me a full-stack web app

"Build a task management app with a React frontend, Express API, and PostgreSQL database."

Mayor creates 8 tasks: 4 for frontend (login page, dashboard, task list, settings), 3 for backend (auth API, task CRUD, database schema), 1 for tests. Dispatches 4 polecats working in parallel. Frontend and backend built simultaneously. Refinery merges everything. You have a working app.

Fix 15 bugs before the release

You paste a bug list. Mayor creates 15 beads, one per bug.

Dispatches workers in waves. Each bug gets fixed in its own branch. Refinery merges each fix, running tests after each one. 15 bugs, resolved in parallel, no conflicts.

Refactor the database layer

"Refactor our ORM layer to use the repository pattern."

Mayor analyzes dependencies and creates a phased plan: Phase 1 (schema changes) must finish before Phase 2 (update queries) which must finish before Phase 3 (update API handlers). Sets up dependency chains. Polecats work each phase. Refinery ensures ordering. Clean refactor without breaking anything.
