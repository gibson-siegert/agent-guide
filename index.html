The Basics

What is Gas Town?

Gas Town is a system that lets you run many AI agents at the same time, all working on your code. Instead of one AI assistant doing one thing at a time, you have an entire team.

01

You give the order

Tell the Mayor what you want in plain English. "Build me a login page." "Fix these 12 bugs." "Add unit tests for the API." The Mayor is your project manager — it listens, plans, and delegates.

02

Work gets broken down

The Mayor splits your request into individual tasks called "beads" (like tickets in Jira). Each bead is a focused, well-scoped piece of work that one agent can handle independently.

03

Workers build in parallel

Workers (called Polecats) each pick up a task and code it in their own isolated copy of your project. Five workers can build five features at the same time — no one steps on anyone else's toes.

04

Code merges safely

When workers finish, the Refinery checks every piece of code before merging it. Conflicts are caught and resolved. Nothing broken ever lands on your main branch.

05

Monitors keep everything healthy

The Witness watches every worker in each project. If one gets stuck, it flags the Mayor who reassigns the work. The Deacon runs health checks across the whole system. If an agent crashes, its work is saved and automatically resumed when it restarts.

The Real Problem

Why Gas Town?

AI coding tools gave you a faster pair of hands. Gas Town gives you an entire engineering department. The difference is not speed — it's leverage.

You are the bottleneck

Right now, every task waits for you. You prompt an agent, wait for output, review it, prompt again. You are a single thread in a world that demands concurrency. Gas Town removes you from the critical path. You describe the goal once, and up to 30 workers execute simultaneously.

Work survives everything

Your laptop sleeps. An agent crashes. Your internet drops. It does not matter. Every piece of in-progress work is saved on a persistent hook. When the system restarts, every agent picks up exactly where it left off. Nothing is lost. Nothing needs to be re-explained.

No more "what's the status?"

Every task is a tracked bead with a lifecycle: open, in progress, blocked, closed. Dependencies are explicit. You always know what is done, what is stuck, and what is next. This is not a chat history you scroll through — it is a real project management system.

Merge safety is automatic

Five agents building five features at once creates merge chaos in any normal workflow. The Refinery solves this. It queues completed work, checks for conflicts, and merges cleanly into your main branch. You never resolve a merge conflict by hand.

Agents manage agents

You do not babysit workers. The Witness watches every agent in every project. If one stalls, it escalates to the Mayor. If the Mayor is stuck, the Deacon intervenes. The system heals itself. Your attention is only needed for decisions that require human judgment.

Any repo, instantly

One command wraps any git repository as a managed rig. Your existing code stays where it is. Your existing workflow stays intact. Gas Town layers on top — it does not replace your tools, it multiplies them.

You Are Probably Here

You are not using it yet

Most people install Gas Town and then keep talking to the Mayor like it is a chatbot. That is like hiring a general contractor and then handing them a hammer. Here is what you are probably doing wrong, and what changes when you stop.

You are doing the work yourself

"Fix this bug." Then you wait. Then you review. Then you say "now fix the next one." One at a time. Sequentially. Like you do not have a workforce.

The unlock: describe all the work at once. "Here are 12 bugs. Fix them all." The Mayor creates 12 beads, dispatches polecats in parallel, and the Refinery merges each fix as it lands. You come back to 12 closed tickets and a clean main branch. Your role shifts from doing the work to describing the work.

Your rigs are docked

You added your project but never undocked it. The Mayor has no agents to dispatch to. It is a coordinator with no one to coordinate.

The unlock: gt rig undock my-project and gt rig start my-project. Now the Witness and Refinery are running. Now when you sling a bead, a polecat spawns, does the work in an isolated worktree, and the Refinery merges it. The entire pipeline is active. Without undocking, you have a tool. After undocking, you have a team.

You are not using convoys

You create tasks one at a time and track them in your head. You lose context. You forget what depends on what.

The unlock: convoys group related beads into a single tracked batch. "Refactor the auth system" becomes a convoy with phased dependencies — schema changes before query updates before API handlers. The Mayor dispatches each phase in order, and gt convoy status shows you the whole picture at a glance. You stop managing tasks and start managing outcomes.

You are still sitting at the terminal

You give the Mayor a task, then watch it work. You wait for it to finish before giving the next instruction. You are present for every step.

The unlock: Gas Town is designed to run without you. Sling the work, close your laptop, go to dinner. The Witness monitors workers. The Refinery merges results. The Mayor handles escalations. When you come back, run gt status and gt convoy list to see what shipped while you were gone. The system does not need your attention. It needs your intent.

The Cast

Meet the team

Every role in Gas Town has a specific job. Here's who does what.

Mayor

Your project manager. Breaks work into tasks, assigns workers, and reports back to you.

Polecats

The workers. Each one picks up a task and codes it in their own isolated branch.

Witness

Per-project monitor. Watches workers and flags any that get stuck or go off-track.

Refinery

The merge queue. Checks code quality and combines everything safely into your main branch.

Deacon

Town-level watchdog. Runs health checks across all projects and the whole system.

Crew

Persistent collaborators (like you). Can guide work, review code, and steer direction.

Get Running

Quickstart

From zero to a working AI workforce in five steps. Each one takes about a minute.

Install

Install Gas Town and its dependencies, then initialize your workspace.

# Install Gas Town and dependencies brew install gastown brew install tmux # Initialize your workspace gt install ~/gt --git

Start it up

Boot the entire Gas Town system with a single command.

cd ~/gt gt up

Add your project

Point Gas Town at any git repository. It becomes a managed "rig."

# Add any git repo as a "rig" gt rig add my-project ~/path/to/your/repo # Wake it up and start its agents gt rig undock my-project gt rig start my-project

Talk to the Mayor

Attach to your AI project manager and describe what you want in plain English.

# Attach to your AI project manager gt mayor attach # Now just talk in plain English: # "Build me a login page with email and password" # "Fix all the CSS spacing issues across the site" # "Add unit tests for the API endpoints"

Watch it work

Monitor your AI workforce as it builds, fixes, and ships code.

gt status # See everything running gt convoy list # Track progress on all work gt polecat list # See all active workers
